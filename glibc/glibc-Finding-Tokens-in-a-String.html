5.10. Dizgeciklere Bölme

Hemen her uygulamada, bir komut dizgesini dizgeciklere ayırmak gibi bazı basit ayrıştırma işlemlerine ihtiyaç duyulur. Bunun için string.h başlık dosyasında bildirilmiş olan strtok işlevi ile kullanılabilir.

      işlev
#include <string.h>
char *strtok(char *restrict  newstring, const char *restrict  delimiters)
Evresel Güvenilirlik: | MT-Unsafe race:strtok | AS-Unsafe | AC-Safe

Bir dizge strtok işlevi peşpeşe çağrılarak dizgeciklerine bölünebilir.

Bölünecek dizge yalnızca ilk çağrıda newstring bağımsız değişkeni ile işleve aktarılır. strtok işlevi bunu bazı dahili durum bilgilerini ayarlamakta kullanır. Sonraki çağrılarda, newstring bağımsız değişkeni ile boş gösterici aktarılması aynı dizgeden başka dizgeciklerin alınacağını belirtir. newstring bağımsız değişkenine boş gösterici olmayan bir gösterici belirterek yapılan her strtok çağrısı, durum bilgilerini yeniden ilklendirir. Hiçbir kütüphane işlevinin yazılımcının bilgisi dahilinde olmayan (bu dahili durum bilgisini karıştıran) strtok çağrısı yapmayacağı garanti edilmiştir.

delimiters bağımsız değişkeni çıkarılacak dizgecikleri belirlemede kullanılan ayraçlardan oluşan bir dizgedir. Bu ayraçlardan birine rastlandığında bu karakter boş bayt ile değiştirilir ve dizgenin başlangıcı newstring bağımsız değişkeninde döndürülür. Dizgecikler daima dizgeciğin sonunda bulunan bir ayracın varlığına göre ayrıştırılır.

Sonraki strtok çağrılarında arama önceki dizgeciği sonlandıran boş bayttan sonraki karakterden başlar. Sonraki strtok çağrılarında hep aynı delimiters dizgesini kullanmak zorunluluğu yoktur.

newstring dizgesinin sonuna gelindiğinde ya da kalan dizge sadece ayraç karakterlerinden oluşuyorsa işlev boş gösterici ile döner.

Çok baytlı dizgelerde birden fazla bayttan oluşan karakterlerin baytları ayrı ayrı ele alınmaz. Bu işlevde baytlar ayrı ayrı ele alınır. İşlev yerelden bağımsızdır.

      işlev
#include <wchar.h>
wchar_t *wcstok(wchar_t *newstring, const wchar_t *delimiters, wchar_t **save_ptr)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bir dizge wcstok işlevi peşpeşe çağrılarak dizgeciklerine bölünebilir.

Bölünecek dizge yalnızca ilk çağrıda newstring bağımsız değişkeni ile işleve aktarılır. wcstok işlevi bunu bazı dahili durum bilgilerini ayarlamakta kullanır. Sonraki çağrılarda, newstring bağımsız değişkeni ile boş gösterici aktarılması aynı dizgeden başka dizgeciklerin alınacağını belirtir. Bölünecek dizgenin başlangıcı olarak daha önce save_ptr dizisine kaydedilen gösterici kullanılır.

delimiters bağımsız değişkeni çıkarılacak dizgecikleri belirlemede kullanılan ayraçlardan oluşan geniş dizgedir. Bu ayraçlardan birine rastlandığında bu karakter boş geniş karakter ile değiştirilir. delimiters üyesi olmayan ilk geniş karakter, yeni dizgeciğin başlangıcını işaretler. Yeni dizgeciğin sonu, delimiters üyesi olan bir sonraki geniş karakter aranarak bulunur. newstring dizgesindeki delimiters üyesi geniş karakterin üzerine boş geniş karakter yazılır, üzerine yazılan geniş karakteri geçen gösterici (bir sonraki aramanın başlangıcı) save_ptr ile ve dizgeciğin başlangıcına gösterici newstring bağımsız değişkeninde döndürülür.

Sonraki wcstok çağrısında arama, önceki dizgeciği sonlandıran boş geniş karakterden sonraki geniş karakterde başlar. Sonraki wcstok çağrılarında hep aynı delimiters dizgesini kullanmak zorunlu değildir.

newstring dizgesinin sonuna gelindiğinde ya da kalan dizge sadece ayraç karakterlerinden oluşuyorsa işlev boş gösterici ile döner.

Uyarı

strtok ve wcstok işlevleri ayrıştırma sırasında dizgeyi değiştirdiğinden, strtok/wcstok çağrısından önce dizge daima geçici bir tampona kaydedilmelidir (Bkz. Dizilerin ve Dizgelerin Kopyalanması). strtok veya wcstok işlevinin yazılımın başka bir parçasından gelen bir dizgeyi değiştirmesine izin verilirse, strtok veya wcstok çağrısından sonra dizge değişmiş olacağından dizgenin başka amaçlar için kullanılması gerektiğinde beklenen değerde olmaması sorunlara sebep olabilir.

Üzerinde işlem yapılan dizge bir sabit olduğu takdirde, strtok veya wcstok işlevi sabiti değiştirmeye kalktığında yazılım salt-okunur belleğe yazacağından bir ölümcül hata oluşacaktır Bkz. ???. strtok veya wcstok işlevinin yaptığı işlem dizgeyi değiştirmeyeceği varsayılsa bile (örneğin, yalnızca bir dizgecik vardır) dizge değiştirilebilir (GNU C Kütüphanesi değiştirecektir).

Bu, genel bir kuralın özel bir durumudur: Eğer bir yazılım parçası belli bir veri yapısını güncelleme ile görevlendirilmemişse sözkonusu veri yapısının bu yazılım tarafından geçici olarak güncellenmesi hatalara yol açabilir.

wcstok işlevinin tersine strtok işlevi çok evreliliğe uygun değildir. Çok evreliliğe uygunluğun niçin ve nerede önemli olduğu ??? bölümünde incelenmiştir.

Örnek: strtok kullanımı
#include <string.h>
#include <stddef.h>

…

const char string[] = "words separated by spaces - and, punctuation!";
const char delimiters[] = " .,;:!-";
char *token, *cp;

…

cp = strdupa (string);                /* yazılabilir bir kopya oluştur.  */
token = strtok (cp, delimiters);      /* token => "words" */
token = strtok (NULL, delimiters);    /* token => "separated" */
token = strtok (NULL, delimiters);    /* token => "by" */
token = strtok (NULL, delimiters);    /* token => "spaces" */
token = strtok (NULL, delimiters);    /* token => "and" */
token = strtok (NULL, delimiters);    /* token => "punctuation" */
token = strtok (NULL, delimiters);    /* token => NULL */

GNU C Kütüphanesi bir dizgeyi dizgeciklerine bölmek için, çok evreliliğe uygun olmamaktan kaynaklanan sınırlamaları aşan iki işlev daha içerir. Bu işlevler geniş dizgeler üzerinde kullanılmaz.

      işlev
#include <string.h>
char *strtok_r(char *newstring, const char *delimiters, char **save_ptr)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bu işlev, strtok gibi, ardışık olarak çağrılarak bir dizgeyi dizgeciklerine ayırmakta kullanılır. Farkı, sonraki dizgecik hakkındaki bilgilerin, wcstok işlevindeki gibi bir dizge göstericisine gösterici olan save_ptr bağımsız değişkeni ile gösterilen alanda saklanmasıdır. İşlevin, newstring bağımsız değişkeninin boş gösterici ile çağrılması ve çağrılar arasında save_ptr bağımsız değişkeninin değişmeden bırakılması, çok evreliliği aksatmaksızın işlemin yapılmasını sağlar.

Bu işlev POSIX.1 içinde tanımlıdır ve çok evreliliği destekleyen çoğu sistemde bulunur.

      işlev
#include <string.h>
char *strsep(char **string_ptr, const char *delimiter)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bu işlev, strtok_r işlevindeki newstring bağımsız değişkeninin yerini string_ptr bağımsız değişkeninin alması dışında strtok_r işlevine benzer. Taşıyıcı göstericinin ilklendirilmesi yazılımcı tarafından yapılır. strsep işlevine art arda yapılan çağrılar, göstericiyi delimiter ile tahsis edilmiş dizgecikler boyunca hareket ettirerek bir sonraki dizgeciğin adresini döndürürken string_ptr bağımsız değişkenini bir sonraki dizgeciğin başlangıcını gösterecek şekilde günceller.

strsep ile strtok_r arasındaki bir diğer fark da, eğer girdi dizgesi içinde delimiter karakterlerinden biri birden fazla içeriliyorsa, her delimiter karakteri çifti için bir boş dizge döndürmesidir. Yani, yazılımın normalde işlevi çalıştırmadan önce işlevin boş dizge döndürüp döndürmeyeceğini sınaması gerekir.

Bu işlev 4.3 BSD içinde tanımlıdır ve genişçe bir kullanım alanı vardır.

Örnek: Önceki örneğin strsep sürümü
#include <string.h>
#include <stddef.h>

…

const char string[] = "words separated by spaces - and, punctuation!";
const char delimiters[] = " .,;:!-";
char *running;
char *token;

…

running = strdupa (string);
token = strsep (&running, delimiters);    /* token => "words" */
token = strsep (&running, delimiters);    /* token => "separated" */
token = strsep (&running, delimiters);    /* token => "by" */
token = strsep (&running, delimiters);    /* token => "spaces" */
token = strsep (&running, delimiters);    /* token => "" */
token = strsep (&running, delimiters);    /* token => "" */
token = strsep (&running, delimiters);    /* token => "" */
token = strsep (&running, delimiters);    /* token => "and" */
token = strsep (&running, delimiters);    /* token => "" */
token = strsep (&running, delimiters);    /* token => "punctuation" */
token = strsep (&running, delimiters);    /* token => "" */
token = strsep (&running, delimiters);    /* token => NULL */

basename işlevinin GNU sürümü, filename ile belirtilen dosya yolunun son elemanı ile döner. filename bağımsız değişkenini içerdiği ters eğik çizgiler bakımından değiştirmediğinden kullanımı tercih edilir. İşlev string.h başlık dosyasında bildirilmiştir. libgen.h başlık dosyası da içerildiğinde bu işlevin XPG sürümü ile değiştirileceği unutulmamalıdır.

Taşınabilirlik Bilgisi

Bu işlev, farklı sistemlerde farklı sonuçlar üretebilir.

      işlev
#include <libgen.h>
char *basename(char *path)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

basename işlevinin XPG sürümüdür ve ruhen GNU sürümüne benzer. Fakat, path içindeki '/' karakterleri silinerek değişikliğe uğratılır. Eğer bağımsız değişken sadece '/' karakterinden oluşuyorsa '/' karakteri döner. Ayrıca, NULL ise ya da boş dizge ise "." döner. İşlevin XPG sürümü libgen.h başlık dosyasında bildirilmiştir.

Örnek: XPG basename kullanımı

main (int argc, char *argv[])
{

  char *path = strdupa (argv[0]);

  prog = basename (path);

  if (argc < 2)
    {
      fprintf (stderr, "Kullanımı: %s <arg>\n", prog);
      exit (1);

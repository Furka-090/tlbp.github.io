17.9. Sözde Uçbirimler

Sözde uçbirim, uçbirim gibi davranan özel bir süreçler arası iletişim kanalıdır. Kanalın bir ucu ana uç ya da sözde ana uçbirim aygıtı olarak adlandırılırken diğer uç yardımcı uç adını alır. Ana uca yazılan veri yardımcı uç tarafından sıradan bir uçbirime kullanıcı tarafından yazılmış gibi alınır ve yardımcı uca yazılan veri ana uca sıradan bir uçbirime yazılmış gibi gönderilir.

Sözde uçbirimler genellikle xterm ve emacs gibi uygulamalar tarafından uçbirim benzetimi amacıyla kullanılır.

17.9.1. Sözde Uçbirimlerin Tahsisi

Bu bölümde sözde uçbirim tahsisi için kullanılan işlevlerden bahsedilecektir. Bu işlevler stdlib.h başlık dosyasında bildirilmiştir.

      işlev
#include <stdlib.h>
int getpt(void)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe fd

getpt işlevi kullanılabilir ilk sözde ana uçbirim aygıtına yeni bir dosya tanıtıcı ile döner. İşlevin normal dönüş değeri negatif olmayan bir tamsayı olarak dosya tanıtıcıdır. Bir hata oluşması durumunda -1 döner. Aşağıdaki errno hata durumları bu işlev için tanımlanmıştır:

ENOENT

Kullanılabilir sözde ana uçbirim aygıtı yok.

Bu işlev GNU oluşumudur.

      işlev
#include <stdlib.h>
int grantpt(int filedes)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe dlopen plugin heap lock | AC-Unsafe corrupt lock fd mem

grantpt işlevi filedes dosya tanıtıcısı ile ilişkili sözde ana uçbirim aygıtının diğer ucu olan yardımcı sözde uçbirimin sahipliğini ve erişim izinlerini değiştirir. Uçbirimin sahibi, işlevi çağıran sürecin gerçek kullanıcı kimliği olur (bkz, ???). Erişim yetkileri ise sahibi tarafından yazılabilir/okunabilir ve sadece grubu tarafından okunabilir olarak ayarlanır.

Bazı sistemlerde bu işlev özel bir setuid root yazılım çağrılarak gerçeklenir (bkz, ???). Dolayısıyla, SIGCHLD sinyali için bir sinyal işleyici oluşturulması grantpt çağrısı ile çelişebilir (bkz, İş Denetim Sinyalleri).

İşlevin normal dönüş değeri sıfırdır. Bir hata oluşması durumunda -1 döner. Aşağıdaki errno hata durumları bu işlev için tanımlanmıştır:

EBADF

filedes geçerli bir dosya tanıtıcı değil.

EINVAL

filedes sözde uçbirim ile ilişkili değil.

EACCES

filedes ile ilişkili olan sözde ana uçbirimin karşı ucu olan sözde yardımcı uçbirime erişilemiyor.

      işlev
#include <stdlib.h>
int unlockpt(int filedes)
Evresel Güvenilirlik: | MT-Safe | AS-Unsafe heap/bsd | AC-Unsafe fd mem

unlockpt işlevi filedes dosya tanıtıcısı ile ilişkili sözde ana uçbirim aygıtının diğer ucu olan sözde yardımcı uçbirimin kilidini kaldırır. Bazı sistemlerde, sözde yardımcı uçbirim sadece kilidi kaldırıldığında açılabilir, bu nedenle taşınabilir uygulamaların sözde yardımcı uçbirimi açarken daima bir unlockpt çağrısı yapması gerekir.

İşlevin normal dönüş değeri sıfırdır. Bir hata oluşması durumunda -1 döner. Aşağıdaki errno hata durumları bu işlev için tanımlanmıştır:

EBADF

filedes geçerli bir dosya tanıtıcı değil.

EINVAL

filedes sözde ana uçbirim ile ilişkili değil.

      işlev
#include <stdlib.h>
char *ptsname(int filedes)
Evresel Güvenilirlik: | MT-Unsafe race:ptsname | AS-Unsafe heap/bsd | AC-Unsafe fd mem

filedes dosya tanıtıcısı sözde ana uçbirim aygıtı ile ilişkili ise, ptsname işlevi ilişkili sözde yardımcı uçbirim aygıtının dosya ismini, boş karakter sonlandırmalı duruk ayrılmış bir dizgeye gösterici olarak döndürür. Bu dizge daha sonraki ptsname çağrıları ile geçersiz kılınabilir.

      işlev
#include <stdlib.h>
int ptsname_r(int filedes, char *buf, size_t *len)
Evresel Güvenilirlik: | MT-Safe | AS-Unsafe heap/bsd | AC-Unsafe fd mem

ptsname_r işlevi ptsname işlevine benzemekle birlikte, sonucu kullanıcı tarafından belirtilen len uzunluktaki buf içinde döndürmesi ile farklıdır.

Bu işlev GNU oluşumudur.

Örnek:

Bu işlevlerin genel kullanım biçimi örnekte gösterilmiştir: 

int
open_pty_pair (int *amaster, int *aslave)
{
  int master, slave;
  char *name;

  master = getpt ();
  if (master < 0)
    return 0;

  if (grantpt (master) < 0 || unlockpt (master) < 0)
    goto close_master;
  name = ptsname (master);
  if (name == NULL)
    goto close_master;

  slave = open (name, O_RDWR);
  if (slave == -1)
    goto close_master;

  *amaster = master;
  *aslave = slave;
  return 1;

close_slave:
  close (slave);

close_master:
  close (master);
  return 0;
}

17.9.2. Sözde Uçbirim Çiftinin Açılması

Bu işlevler BSD'den alınmıştır, ayrı bir kütüphane olarak libutil kütüphanesinde bulunur ve pty.h başlık dosyasında bildirilmiştir.

      işlev
#include <pty.h>
int openpty(int *amaster, int *aslave, char *name, const struct termios *termp, const struct winsize *winp)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe dlopen plugin heap lock | AC-Unsafe corrupt lock fd mem

Bu işlev sözde uçbirim çiftini ayırdıktan sonra açar ve sözde ana uçbirimin dosya tanıtıcısını *amaster göstericisinde, sözde yardımcı uçbirimin dosya tanıtıcısını *aslave göstericisinde döndürür. name bağımsız değişkeni boş gösterici değilse, sözde yardımcı uçbirim aygıtının dosya ismi *name içinde saklanır. termp boş gösterici değilse, sözde yardımcı uçbirimin öznitelikleri termp ile gösterilen yapıdan ayarlanır (bkz. Uçbirim Kipleri). Benzer şekilde, winp boş gösterici değilse, sözde uçbirimin ekran boyutları winp ile gösterilen yapıdan ayarlanır.

İşlevin normal dönüş değeri sıfırdır. Bir hata oluşması durumunda -1 döner. Aşağıdaki errno hata durumları bu işlev için tanımlanmıştır:

EBADF

Kullanılabilir sözde uçbirim çifti yok.

Uyarı

openpty işlevinin name bağımsız değişkeninin NULL olması çok tehlikelidir, çünkü işlev name dizgesinin taşmasına karşı korunmamıştır. Sözde yardımcı uçbirim aygıtının ismini bulmak için işlevdeki bağımsız değişken yerine, ttyname işlevinin openpty tarafından döndürülen *aslave dosya tanıtıcısı ile kullanılması gerekir.

      işlev
#include <pty.h>
int forkpty(int *amaster, char *name, const struct termios *termp, const struct winsize *winp)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe dlopen plugin heap lock | AC-Unsafe corrupt lock fd mem

Bu işlev openpty işlevine benzer, ek olarak yeni bir süreç çatallar ve yeni açılan sözde yardımcı uçbirim aygıtını alt sürecin denetim uçbirimi yapar.

İşlem başarılı olursa hem çağıran süreç hem de alt süreç işlevin döndüğünü görür fakat işlev farklı değerlerle döner: alt süreçte 0 değeriyle dönerken, çağıran sürece alt sürecin süreç kimliğini döndürür.

Sözde uçbirim çifti tahsis edilememişse ya da alt süreç oluşturulamamışsa işlev başarısız olmuş demektir, bu durumda işlev çağrıldığı sürece -1 değerini döndürür.

Uyarı

forkpty işlevinde de openpty gibi name bağımsız değişkeni sorunu vardır.

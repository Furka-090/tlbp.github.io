24.6.1. Kendine Sinyal Göndermek

Bir sürecin kendine sinyal göndermesi için raise işlevi kullanılır. Bu işlev signal.h başlık dosyasında bildirilmiştir.

      işlev
#include <signal.h>
int raise(int signum)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

raise işlevi çağrıldığı sürece signum sinyalini gönderir. İşlem başarılı olursa sıfırla, aksi takdirde sıfırdan farklı bir değerle döner. Başarısızlığın tek sebebi signum değerinin geçersiz olmasıdır.

      işlev
#include <signal.h>
int gsignal(int signum)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

gsignal işlevi raise ile aynı işi yapar; sadece SVID uyumluluğu için vardır.

raise işlevinin genellikle kullanıldığı tek yer yakalanan sinyalin ön tanımlı davranışını yeniden üretmektir. Örneğin, yazılımın bir kullanıcı SUSP karakterini (C-z; bkz. ???) tuşlamış olsun. Bu işlem yazılıma etkileşimli durdurma sinyalinin (SIGTSTP) gönderilmesine sebep olur. Bu durumda durdurma öncesi bazı tamponların temizlenmesi gerekir.

Örnek: 
#include <signal.h>

/* Bir durdurma sinyali geldiğinde, eylemi önce
   ön tanımlı eyleme ayarla, temizliği yaptıktan
   sonra sinyali yeniden gönder. */

void
tstp_isleyici (int sig)
{
  signal (SIGTSTP, SIG_DFL);
  /* Burada temizlik işlemleri var. */
  …
  raise (SIGTSTP);
}

/* Süreç çalışmaya kaldığı yerden devam
   edeceği zaman sinyal işleyiciyi yeniden kur. */

void
cont_isleyici (int sig)
{
  signal (SIGCONT, cont_isleyici);
  signal (SIGTSTP, tstp_isleyici);
}

/* Yazılım başlatıldığında iki işleyiciyi de etkinleştir. */

int
main (void)
{
  signal (SIGCONT, cont_isleyici);
  signal (SIGTSTP, tstp_isleyici);
  …
}

Taşınabilirlik Bilgisi

raise işlevi ISO C komisyonu tarafından tasarlanmıştır. Daha eski sistemler işlevi desteklemeyebilir, bu bakımdan kill işlevinin kullanılması daha taşınabilir olacaktır. Bkz. Başka Bir Sürece Sinyal Göndermek.

7.9. Evet/Hayır Yanıtları

Çizgesel arayüzü olmayan bazı uygulamalarda evet ya da hayır olarak yanıtlanması gereken sorular olur. Eğer iletiler yabancı dillere çevriliyorsa, yanıtlarında yerelleştirilmesi gerekir. Soruyu bir dilde sorup, başka bir dilde (genellikle ingilizce) yanıt istemek tuhaf olur.

GNU C Kütüphanesi uygulamaların ilgili yerel tanımına kolayca erişebilmesini mümkün kılan rpmatch işlevini içerir.

      işlev
#include <stdlib.h>
int rpmatch(const char *response)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe corrupt heap lock dlopen | AC-Unsafe corrupt lock mem fd

rpmatch işlevi response dizgesinde evet ya da hayır dizgelerinden birinin varlığını arar. Bu işlem seçili yerelin LC_MESSAGES yerel sınıfının YESEXPR ve NOEXPR düzenli ifadeleri ile response dizgesi karşılaştırılarak yapılır. İşlev şu değerlerden biri ile döner:

1

Kullanıcı olumlu yanıt verdi.

0

Kullanıcı olumsuz yanıt verdi.

-1

Yanıt ne YESEXPR ne de NOEXPR düzenli ifadesine uyuyor.

Bu işlev standartlaşmamıştır ama GNU C Kütüphanesinde ve IBM AIX kütüphanesinde bulunmaktadır.

Bu işlev normalde şöyle kullanılabilir:

Örnek: 
  …
  /* Öntanımlı bir değer olsun.  */
  _Bool doit = false;

  fputs (gettext ("Do you really want to do this? "), stdout);
  fflush (stdout);
  /* getline çağrısını hazırla.  */
  yanit = NULL;
  len = 0;
  while (getline (&line, &len, stdin) >= 0)
    {
      /* Yanıta bak.  */
      int res = rpmatch (yanit);
      if (res >= 0)
        {
          /* Yanıt olumluysa.  */
          if (res > 0)
            doit = true;
          break;
        }
    }
  /* Yanıt tamponunu serbest bırak.  */
  free (yanit);

Döngünün bir hata ya da olumlu veya olumsuz bir yanıt alınıncaya kadar sürdürüldüğüne dikkat edilmelidir.

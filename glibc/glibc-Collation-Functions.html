5.8. Dizgeleri Yerele Özgü Karşılaştırma İşlevleri

Bazı yerellerde abecesel sıralama karakter kodlarının sıralamasından farklıdır. Örneğin, Türkçedeki abecesel "CÇD" sıralaması 67-199-68 karakter kodu sıralamasından farklıdır. Harfler sırf karakter koduna göre sıralansaydı, bazıları sıralamada sona kalacaktı.

string.h başlık dosyasında bildirilen strcoll ve strxfrm işlevleri ile wchar.h başlık dosyasında bildirilen wcscoll ve wcsxfrm işlevleri, yerele özgü karakter sıralamasına uygun dizge karşılaştırmalarında kullanılabilir. Bu işlevlerin kullanılacağı yerel, LC_COLLATE yerel sınıfına gerekli değer atanarak belirtilebilir. Daha ayrıntılı bilgi için bkz. Yereller ve Uluslararasılaştırma.

Standart C yerelinde karakter sıralaması bakımından strcoll ile strcmp işlevlerinin davranışlarında bir fark yoktur. Benzer olarak wcscoll ve wcscmp işlevleri de bu bakımdan aynıdır.

Bu işlevleri etkin olarak çalıştırmanın yolu çok baytlı bir dizgenin içindeki karakterleri yerelin karakter sıralamasına uygun bir konumlamayla bir bayt sıralamasına dönüştürmektir. Böyle oluşturulmuş bayt sıralamaları ile yerelin karakter sıralamasına uygun olarak dizgeleri karşılaştırmak artık daha kolaydır.

strcoll ve wcscoll işlevleri bu dönüşümü karşılaştırma sırasından dolaylı olarak uygular. strxfrm ve wcsxfrm işlevleri ise tam aksine doğrudan doğruya karakter sıralaması/abecesel sıralama eşleştirmesi yapar. Bir dizge kümesi üzerinde çok sayıda karşılaştırma yapılacaksa önce strxfrm veya wcsxfrm işlevlerini kullanarak dizgeleri bir kerede dönüştürüp ardından strcmp veya wcscmp ile dönüştürülmüş dizgeleri karşılaştırmak daha verimli bir yöntemdir.

Örnek: qsort ile kullanılan karşılaştırma işlevi

Bu örnekte bir dizge dizisi strcoll ile karşılaştırılarak sıralanmaktadır. Burada gerçek sıralama algoritması yazılmamıştır; bunun için qsort (Dizi Sıralama İşlevi) kullanılmıştır. Buradaki kodun yaptığı iş dizgeler sıralanırken nasıl karşılaştırılacağını göstermektir. (Bu bölümün devamında strxfrm kullanarak bunun daha verimli olarak nasıl yapılacağından ayrıca bahsedilecektir.) 

int
elemanlari_karsilastir (const void *v1, const void *v2)
{
  char * const *p1 = v1;
  char * const *p2 = v2;

  return strcoll (*p1, *p2);
}

/* Burası giriş noktası — yerelin karakter sıralamasını
   kullanarak dizgeleri sıralayan işlev. */

void
sirala (char **dizi, int dizge_sayisi)
{
  /* Dizgeleri karşılaştırarak diziyi sırala. */
  qsort (dizi, dizge_sayisi,
         sizeof (char *), elemanlari_karsilastir);
}

from dizgesinin to dizgesine sarkması durumundaki davranış belirlenmemiştir (bkz. Dizilerin ve Dizgelerin Kopyalanması).

Dönen değer dönüştürülen dizgenin uzunluğudur. Bu değer size değerinden etkilenmez, ancak size bayttan büyük ya da eşitse, dönüştürülen dizge to dizisine sığmamış, diziye dizgenin sığdığı kadarı kaydedilmiş demektir. Dönüştürülen dizgenin tamamını almak için işlevin daha büyük bir dizi ile tekrar çağrılması gerekir.

Örnek: struct siralayici dizisini sıralamak için qsort ile kullanılan karşılaştırma işlevi

Buradaki örnekte, çok sayıda karşılaştırma yapılması durumunda strxfrm işlevinin nasıl kullanılacağı gösterilmiştir. Önceki örnekle aynı şeyi yapsa da buradaki daha hızlıdır, çünkü her dizgeye sadece bir kere dönüşüm uygular, diğer dizgelerle kaç defa karşılaştırma yapıldığının önemi yoktur. Çok sayıda dizge olduğunda kazanılan zaman, bellek tahsisi ve serbest bırakılması için harcanan zamandan bile daha büyüktür. 

struct siralayici { char *girdi; char *donusmus; };

int
elemanlari_karsilastir (const void *v1, const void *v2)
{
  const struct siralayici *p1 = v1;
  const struct siralayici *p2 = v2;

  return strcmp (p1->donusmus, p2->donusmus);
}

/* Burası giriş noktası — yerelin karakter sıralamasını
   kullanarak dizgeleri sıralayan işlev. */

void
hizli_sirala (char **dizi, int dizge_sayisi)
{
  struct siralayici gecici_dizi[dizge_sayisi];
  int i;

  /* gecici_dizi  dizisini ilklendir.
     Her eleman bir girdi dizgesi ve onun dönüştürülmüşünü içersin. */
  for (i = 0; i < dizge_sayisi; i++)
    {
      size_t uzunluk = strlen (dizi[i]) * 2;
      char *donusmus;
      size_t donusmus_uzunluk;

      gecici_dizi[i].girdi = dizi[i];

      /* Önce yeterince büyük bir tamponla dene.  */
      donusmus = (char *) xmalloc (uzunluk);

      /* dizi[i] dizisini dönüştür.  */
      donusmus_uzunluk = strxfrm (donusmus, dizi[i], uzunluk);

      /* Tampon yetersizse yeniden boyutlandırıp tekrar dene.  */
      if (donusmus_uzunluk >= uzunluk)
        {
          /* Gerekli belleği tahsis et.
             Sonlandırıcı boş bayt için +1'i unutma.  */
          donusmus = xrealloc (donusmus,
                                  donusmus_uzunluk + 1);

          /* Dönen değerin önemi yok çünkü dönüşmüş
             dizgenin uzunluğu biliniyor.  */
          (void) strxfrm (donusmus, dizi[i],
                          donusmus_uzunluk + 1);
        }

      gecici_dizi[i].donusmus = donusmus;
    }

  /* Dönüştürülmüş dizgeleri karşılaştırarak geçici diziyi sırala. */
  qsort (gecici_dizi, dizge_sayisi,
         sizeof (struct siralayici), elemanlari_karsilastir);

  /* Elemanları geri, kalıcı diziye sıralı olarak yerleştir. */
  for (i = 0; i < dizge_sayisi; i++)
    dizi[i] = gecici_dizi[i].girdi;

  /* tahsis edilen dizgeleri serbest bırak. */
  for (i = 0; i < dizge_sayisi; i++)
    free (gecici_dizi[i].donusmus);
}

Uyumluluk Bilgisi

Dizgelerin yerele özgü harf sıralama işlevleri ISO C90'nın yeni bir özelliğidir. Daha eski C oluşumlarında buna eşdeğer bir özellik yoktur. Geniş karakter sürümü ise ISO C90'nın 1. düzeltmesinde yer almıştır.

22.3.4.2. Geleneksel Zamanlama İşlevleri

Bu bölümde bir sürecin nezaket değerinin nasıl okunabileceğinden ve nasıl belirtilebileceğinden bahsedilecektir. Bu simgelerin tamamı sys/resource.h başlık dosyasında bildirilmiştir.

İşlev ve makro isimleri POSIX tarafından tanımlanmıştır. POSIX'te ve bu kılavuzda isimlendirme için "öncelik" (priority) terimi kullanılmış olmasına rağmen işlevler aslında nezaket değerleri ile işlem yaparlar.

Geçerli nezaket değerleri çekirdeğe bağlı olmasına rağmen, genellikle -20 ile 20 arasındadır. Daha düşük bir nezaket değeri daha yüksek önceliğe karşılıktır. Öncelik değerlerinin aralığını belirleyen sabitler şunlardır:

      sabit
#include <sys/resource.h>
PRIO_MIN

Geçerli en düşük nezaket değeri.

      sabit
#include <sys/resource.h>
PRIO_MAX

Geçerli en yüksek nezaket değeri.

      işlev
#include <sys/resource.h>
int getpriority(int class, int id)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bir süreç kümesinin nezaket değerini döndürür; class ve id hangisi olduğunu belirtir (aşağıya bakınız). Eğer belirtilen süreçlerin hepsi aynı nezaket değerine sahip değilse, işlev bunların içinden en düşük nezaket değerini döndürür.

Hata oluşmuşsa -1 döner ve errno değişkenine hata durumu atanır, yoksa dönen değer nezaket değeridir. Aşağıdaki errno hata durumları bu işlev için tanımlanmıştır:

ESRCH

class ve id birlikte herhangi bir mevcut süreçle uyuşmuyor.

EINVAL

class değeri geçersiz.

Dönen değer -1 olduğunda, bunun bir başarısızlık mı yoksa bir nezaket değeri mi olduğunu anlayabilmek için tek yol getpriority çağrısından önce errno = 0 ataması yapmaktır. Çağrının ardından errno != 0 ile başarısızlık sınaması yapılabilir.

      işlev
#include <sys/resource.h>
int setpriority(int class, int id, int niceval)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bir süreç kümesinin nezaket değerini niceval değerine ayarlar; class ve id hangisi olduğunu belirler (aşağıya bakınız).

İşlev başarılı olursa 0 ile döner. Hata oluşmuşsa -1 döner ve errno değişkenine hata durumu atanır. Aşağıdaki errno hata durumları bu işlev için tanımlanmıştır:

ESRCH

class ve id birlikte herhangi bir mevcut süreçle uyuşmuyor.

EINVAL

class değeri geçersiz.

EPERM

Çağrı, sahibi çağıran süreçten farklı bir kulllanıcıya ait bir sürecin nezaket değerini değiştirmeye çalışıyor (yani, hedef sürecin gerçek ya da etkin kullanıcı kimliği, işlevi çağıran sürecin ektin kullanıcı kimliği ile aynı değil) ve işlevi çağıran süreç CAP_SYS_NICE yetkisine sahip değil.

EACCES

Çağrı daha düşük bir nezaket değeri isteği yaptı ama sürecin CAP_SYS_NICE yetkisi yok.

class ve id bağımsız değişkenleri birlikte ilgilenilen süreç kümesini belirler. class için olası değerler şunlardır:

      sabit
#include <sys/resource.h>
PRIO_PROCESS

Belirli bir süreç. id sürecin süreç kimliğidir.

      sabit
#include <sys/resource.h>
PRIO_PGRP

Belirli bir süreç grubundaki bütün süreçler. id süreç grup kimliğidir.

      sabit
#include <sys/resource.h>
PRIO_USER

Belirli bir kullanıcıya ait olan tüm süreçler (yani, gerçek kullanıcı kimliği, kullanıcı ile aynı olan süreçler). id kullanıcının kullanıcı kimliğidir.

id değeri sıfırsa, çağıran süreci belirtir, süreç grubu ya da sürecin sahibi class ile belirtilir.

      işlev
#include <sys/resource.h>
int nice(int increment)
Evresel Güvenilirlik: | MT-Unsafe race:setpriority | AS-Unsafe | AC-Safe

Çağıran sürecin nezaket değerini increment kadar artırır. İşlev başarılı olursa yeni nezaket değeri ile döner. -1 dönüş değeri ise bir hata oluştuğunu gösterir. Bu durumda errno değişkenine setpriority işlevindeki değerler atanır.

Örnek:

nice işlevine eşdeğer bir işlev tanımı: 

int
nice (int increment)
{
  int result, old = getpriority (PRIO_PROCESS, 0);
  result = setpriority (PRIO_PROCESS, 0, old + increment);
  if (result != -1)
      return old + increment;
  else
      return -1;
}

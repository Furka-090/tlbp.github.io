5.4. Dizilerin ve Dizgelerin Kopyalanması

Bu kısımda dizi ve dizge ve geniş dizgelerin içeriğini kopyalayan işlevler açıklanmıştır. İsimleri str ve mem ile başlayan işlevler string.h başlık dosyasında, w ile başlayan işlevler ise wchar.h başlık dosyasında bildirilmiştir.

Bu bölümdeki işlevlere ilişkin bağımsız değişkenlerin sırasını hatırlamanın yararlı bir yolu, sıranın bir atama ifadesine karşılık gelmesidir. Bağımsız değişken sırası hedef, kaynak şeklinde belirtilen bu tür işlevlerin çoğu hedef dizinin adresi ile bazıları da hedefin boş sonlandırıcısının konumu veya boş sonlandırıcı yoksa hedefin uzunluğu ile döner.

Kaynak dizi ve hedef diziden uzunsa bu işlevlerin çoğu çalışmaz. Örneğin, kaynak dizisinin sonu, hedef dizisinin başlangıcını aşarsa, kaynak dizisi hedefe yazılırken başlangıcı, kendi sonunun üzerine yazılır. Daha kötüsü, bunların dizge içermesi durumunda kaynak dizgenin sonlandırıcısı kaybolur ve bu durumda oluşan döngü yazılıma tahsis edilen belleğin tamamen çöpe dönüşmesine kadar gidebilir.

Dizilerin kopyalanmasında üst üste binme sorunu olan tüm işlevler bu kılavuzda açıkça belirtilmiştir. Bu kısımdaki işlevlere ek olarak sprintf (???) ve scanf (???) gibi başka işlevler de vardır.

      işlev
#include <string.h>
void *memcpy(void *restrict  to, const void *restrict  from, size_t  size)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

memcpy işlevi size baytı from adresinde başlayan nesneden to adresinde başlayan nesneye kopyalar. Bu işlevde from dizisinin to dizisinin üzerine binmesi durumundaki davranışı tanımsızdır; üst üste binme olasılığı varsa memmove kullanılmalıdır.

memcpy işlevi to değeri ile döner.

Örnek:

Bu örnekte bir dizinin içeriğini kopyalamak için memcpy işlevinin nasıl kullanılacağı gösterilmiştir: 

struct foo *eskidizi, *yenidizi;
int diziboyu;
…
memcpy (yeni, eski, diziboyu * sizeof (struct foo));

      işlev
#include <wchar.h>
wchar_t *wmemcpy(wchar_t *restrict  wto, const wchar_t *restrict  wfrom, size_t  size)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

wmemcpy işlevi size geniş karakteri wfrom adresinde başlayan nesneden wto adresinde başlayan nesneye kopyalar. Bu işlevde wfrom dizisinin wto dizisinin üzerine binmesi durumundaki davranış tanımsızdır; üst üste binme olasılığı varsa wmemmove kullanılmalıdır.

Örnek:

Burada wmemcpy işlevinin olası bir gerçeklemesi vardır. Ancak, daha fazla en iyileme yapılabilir. 

wchar_t *
wmemcpy (wchar_t *restrict wto, const wchar_t *restrict wfrom,
         size_t size)
{
  return (wchar_t *) memcpy (wto, wfrom, size * sizeof (wchar_t));
}

wmemcpy işlevi wto değeri ile döner.

Bu işlev ISO C90 standardının 1. düzeltmesinde tanımlanmıştır.

      işlev
#include <string.h>
void *mempcpy(void *restrict  to, const void *restrict  from, size_t  size)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

mempcpy işlevi memcpy işlevi ile hemen hemen eşdeğerdir. size baytı, göstericisi from olan nesneden göstericisi to olan nesneye kopyalar. Fakat dönen değer to nesnesinin değeri değildir. to nesnesine yazılan son bayttan sonraki baytın göstericisi ile döner. Yani, dönen değer: 

((void *) ((char *) to + size)).

Bu işlev çok sayıda nesnenin ardışık bellek konumlarına kopyalanması durumunda kullanışlıdır.

Örnek: 
void *
combine (void *o1, size_t s1, void *o2, size_t s2)
{
  void *result = malloc (s1 + s2);
  if (result != NULL)
    mempcpy (mempcpy (result, o1, s1), o2, s2);
  return result;
}

Bu işlev GNU oluşumudur.

      işlev
#include <wchar.h>
wchar_t *wmempcpy(wchar_t *restrict  wto, const wchar_t *restrict  wfrom, size_t  size)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

wmempcpy işlevi wmemcpy işlevi ile hemen hemen eşdeğerdir. İşlev, size geniş karakteri wfrom adresinden başlayan nesneden wto adresine kopyalar. Fakat dönen değer wto değeri değildir. wto başlangıcından itibaren yazılan son geniş karakteri izleyen geniş karakterin adresi ile döner. Yani, dönen değer: wto + size.

Bu işlev çok sayıda nesnenin ardışık bellek konumlarına kopyalanması durumunda kullanışlıdır.

Örnek:

Burada wmemcpy işlevinin olası bir gerçeklemesi vardır. Ancak, daha fazla en iyileme yapılabilir. 

wchar_t *
wmempcpy (wchar_t *restrict wto, const wchar_t *restrict wfrom,
          size_t size)
{
  return (wchar_t *) mempcpy (wto, wfrom, size * sizeof (wchar_t));
}

Bu işlev GNU oluşumudur.

      işlev
#include <string.h>
void *memmove(void *to, const void *from, size_t  size)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

memmove işlevi size baytı from’dan to’ya birbirlerinin üstüne binseler bile kopyalar. Üst üste binme durumunda, işlev, to dilimindeki baytlar dahil from dilimindeki baytları to dilimine dikkatle kopyalar.

İşlev, to değeri ile döner.

      işlev
#include <wchar.h>

Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

wmemmove işlevi size baytı wfrom’dan wto’ya üst üste binseler bile kopyalar. Üst üste binme durumunda, işlev, wto dilimindeki baytlar dahil wfrom dilimindeki baytları wto dilimine dikkatle kopyalar.

Örnek:

Burada wmemmove işlevinin olası bir gerçeklemesi vardır. Ancak, daha fazla en iyileme yapılabilir. 

wchar_t *
wmemmove (wchar_t *wto, const wchar_t *wfrom,
          size_t size)

}

İşlev, wto değeri ile döner.

      işlev
#include <string.h>
void *memccpy(void *restrict  to, const void *restrict  from, int  c, size_t  size)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bu işlev size baytlık dizinin içindeki c karakterine kadar olan kısmı from’dan to’ya kopyalar. Dönen değer, to’ya kopyalanan kısmın c karakterini izleyen baytının adresidir, eğer size bayt içinde c karakteri bulunamamışsa işlev boş gösterici ile döner.

#include <string.h>
char *strcpy(char *restrict  to, const char *restrict  from)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

from dizgesini (sonlandırıcı boş bayta kadar) to dizgesine kopyalar. Bu işlevde from dizgesinin to dizgesinin üzerine binmesi durumundaki davranışı tanımsızdır. İşlev, to dizgesi ile döner.

      işlev
#include <wchar.h>
wchar_t *wcscpy(wchar_t *restrict  wto, const wchar_t *restrict  wfrom)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

wfrom dizgesini (sonlandırıcı boş geniş karaktere kadar) wto dizgesine kopyalar. Bu işlevde wfrom dizgesinin wto dizgesinin üzerine binmesi durumundaki davranışı tanımsızdır. İşlev, wto değeri ile döner.

      işlev
#include <string.h>
char *strdup(const char *s)
Evresel Güvenilirlik: | MT-Safe | AS-Unsafe heap | AC-Unsafe mem

Bu işlev s dizgesini yeni tahsis edilmiş bellek alanına kopyalar. Tahsis işlemi malloc ile yapılır; bkz. Kısıtlanmamış Bellek Tahsisi. malloc yeni dizge için bellek tahsis edemezse işlev boş gösterici ile döner, aksi takdirde yeni dizgeye gösterici ile döner.

      işlev
#include <wchar.h>
wchar_t *wcsdup(const wchar_t *ws)
Evresel Güvenilirlik: | MT-Safe | AS-Unsafe heap | AC-Unsafe mem

Bu işlev ws geniş dizgesini yeni tahsis edilmiş bellek alanına kopyalar. Tahsis işlemi malloc ile yapılır; bkz. Kısıtlanmamış Bellek Tahsisi. malloc yeni geniş dizge için bellek tahsis edemezse işlev boş gösterici ile döner, aksi takdirde yeni geniş dizgeye gösterici ile döner.

Bu işlev GNU oluşumudur.

      işlev
#include <string.h>
char *stpcpy(char *restrict  to, const char *restrict  from)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bu işlev to dizgesinin sonunu gösteren bir gösterici ile dönmesi dışında strcpy işlevi gibidir.

Örnek:

Bu yazılımda foo ve bar birleştirilerek foobar üretilip çıktılanmaktadır: 

#include <string.h>
#include <stdio.h>

int
main (void)
{
  char buffer[10];
  char *to = buffer;
  to = stpcpy (to, "foo");
  to = stpcpy (to, "bar");
  puts (buffer);
  return 0;
}

Bu işlev, POSIX.1-2008 ve sonraki sürümlerin bir parçasıdır, ancak standartlaştırılmadan çok önce bir genişletme olarak GNU C Kütüphanesinde ve diğer sistemlerde mevcuttu.

Dizgelerin üst üste binmesi durumundaki davranış tanımsızdır.

      işlev
#include <wchar.h>
wchar_t *wcpcpy(wchar_t *restrict  wto, const wchar_t *restrict  wfrom)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bu işlev wto dizgesinin sonuna gösterici ile dönmesi dışında wcscpy işlevi gibidir. İşlev kopyalanan dizgenin başlangıç adresi ile değil, dizgeyi sonlandıran boş geniş karakterin adresi ile döner.

Bu işlev bir ISO veya POSIX oluşumu değildir, ancak GNU C kütüphanesi geliştirilirken kullanışlı bulunmuştur.

Üst üste binme durumunda işlevin davranışı tanımsızdır. Bu işlev GNU oluşumudur.

      makro
#include <string.h>
char *strdupa(const char *s)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bu makro, yeni dizgeye bellek tahsis etmek için malloc yerine alloca (bkz. Değişken Boyutlu Özdevinimli Saklama) kullanması dışında strdup gibidir. Yani, alloca kullanarak tahsis edilen bellek dilimleri ile aynı sınırlamalara sahip bir dizi döner.

Çeşitli sebeplerle strdupa sadece makro olarak gerçeklenmiştir; yani bir işlev gibi adresi alınamaz. Bu sınırlama dışında işlev olarak kullanışlıdır.

Örnek:

Aşağıdaki kodda malloc kullanıldığında işlemin ne kadar pahalıya malolduğu gösterilmiştir. 

#include <paths.h>
#include <string.h>
#include <stdio.h>

const char path[] = _PATH_STDPATH;

int
main (void)
{
  char *wr_path = strdupa (path);
  char *cp = strtok (wr_path, ":");

  while (cp != NULL)
    {
      puts (cp);
      cp = strtok (NULL, ":");
    }
  return 0;
}

strtok'un doğrudan path kullanılarak çağrılmasının geçersiz oluşuna dikkat edilmelidir. Ayrıca, strdupa bağımsız değişken aktarımında girişime sebep olabilen alloca'yı (bkz. Değişken Boyutlu Özdevinimli Saklama) kullandığından strtok'un bağımsız değişken listesinde strdupa kullanımına izin verilmez.


Bu işlev sadece GNU C derleyicisi ile kullanılabilir.

32.12. Dizge Değerli Sistem Değişkenleri

POSIX.2, işletim sisteminden dizge değerli sistem değişkenlerinin değerlerinin öğrenilmesi için confstr işlevini tanımlamıştır:

      işlev
#include <unistd.h>
size_t confstr(int parameter, char *buf, size_t len)
Evresel Güvenilirlik: | MT-Safe | AS-Safe | AC-Safe

Bu işlev bir dizge değerli sistem değişkeninin değerini okur ve buf'da başlayan bellek alanının len baytına bu dizgeyi yerleştirerek döner. parameter bağımsız değişkeni aşağıda listelenen _CS_ simgelerinden biri olmalıdır.

İşlevin normal dönüş değeri istenen dizgenin uzunluğudur. buf olarak boş dizge verilmişse işlev dizgeyi buraya yerleştirmeye çalışmaz, sadece dizgenin uzunluğu ile döner. 0 dönüş değeri hata oluştuğunu gösterir.

Eğer istenen dizge için tamponda yeterince yer yoksa (yani len - 1'den daha uzunsa), işlev dizgenin ilk len - 1 baytını (sonlandırıcı boş karaktere yer bırakarak) yerleştirir. Bu durumun oluştuğunu, işlev len bayta eşit ya da daha büyük bir değerle dönerek bildirir.

Aşağıdaki errno hata durumu bu işlev için tanımlanmıştır:

EINVAL

parameter sistem değişkeninin değeri geçersiz.

confstr işlevinin okuyabileceği sistem değişkenleri:

   değişken
#include <unistd.h>
_CS_PATH

Çalıştırılabilir dosyaların aranacağı ön tanımlı dosya yollarının önerilen değeridir. Kullanıcı sisteme oturum açtığında ön tanımlı olarak bu dosya yollarına sahip olur.

   değişken
#include <unistd.h>
_CS_LFS_CFLAGS

Eğer bir kaynak _LARGEFILE_SOURCE kullanarak derlenmişse, C derleyiciye hangi ek seçeneklerin verileceğini belirten dizge döner. Bkz. Özellik Sınama Makroları.

   değişken
#include <unistd.h>
_CS_LFS_LDFLAGS

Eğer bir kaynak _LARGEFILE_SOURCE kullanarak derlenmişse, ilintileyiciye hangi ek seçeneklerin verileceğini belirten dizge döner. Bkz. Özellik Sınama Makroları.

   değişken
#include <unistd.h>
_CS_LFS_LIBS

Eğer bir kaynak _LARGEFILE_SOURCE kullanarak derlenmişse, uygulamanın hangi ek kütüphanelerle ilintileneceğini belirten dizge döner. Bkz. Özellik Sınama Makroları.

   değişken
#include <unistd.h>
_CS_LFS_LINTFLAGS

#include <unistd.h>

Örnek:

confstr işlevini, dönecek dizgeye keyfi bir sınır belirtmeden kullanmanın tek yolu işlevi iki kere çağırmaktır. İlk çağrıda dizgenin uzunluğu döner, buna göre tampon ayırıp tamponu doldurması için işlevi tekrar çağırmak gerekir: 

char *
get_default_path (void)
{
  size_t len = confstr (_CS_PATH, NULL, 0);
  char *buffer = (char *) xmalloc (len);

  if (confstr (_CS_PATH, buf, len + 1) == 0)
    {
      free (buffer);
      return NULL;
    }

  return buffer;
}

17.8. Parolaların Okunması

Bir parolayı okurken, gizli kalmasına yardımcı olmak için ekranda görüntülenmesinden kaçınılması önerilir. Aşağıdaki işlev bunu uygun bir şekilde ele alır.

      işlev
#include <unistd.h>
char *getpass(const char *prompt)
Evresel Güvenilirlik: | MT-Unsafe term | AS-Unsafe heap lock corrupt | AC-Unsafe term lock corrupt

getpass komut istemini çıktılar, ardından uçbirimden gelen dizgeyi görüntülemeden okur. Mümkünse, kullanıcıların dosyalardaki düz metin parolaları kullanmamaları için gerçek uçbirim olan /dev/tty'ye bağlanmaya çalışır; aksi takdirde stdin ve stderr kullanır. getpass ayrıca ISIG uçbirim niteliğini kullanarak uçbirimdeki INTR, QUIT ve SUSP karakterlerini de devre dışı bırakır (bkz, Yerel Kipler). Uçbirim, işlevden önce ve sonra temizlenir, böylece yanlış yazılmış bir parolanın karakterleri yanlışlıkla görünmez.

Diğer C kütüphanelerinde, getpass yalnızca parolanın ilk PASS_MAX baytını döndürebilir. GNU C Kütüphanesinin böyle bir sınırı yoktur, dolayısıyla PASS_MAX tanımsızdır.

Bu işlev unistd.h başlık dosyasında bildirilmiştir. PASS_MAX ise limits.h başlık dosyasında tanımlanır.

Bu hassas işlemler dizisi tüm olası durumlara uygun olmayabilir. Bu durumda, yazılımcının kendi getpass işlevini yazması önerilir. Aşağıda çok basit bir örnek yer almaktadır:

Örnek: 
#include <termios.h>
#include <stdio.h>

ssize_t
my_getpass (char **lineptr, size_t *n, FILE *stream)
{
  struct termios old, new;
  int nread;

  /* yankıyı kapat, kapanmıyorsa başarısız ol. */
  if (tcgetattr (fileno (stream), &old) != 0)
    return -1;
  new = old;
  new.c_lflag &= ~ECHO;
  if (tcsetattr (fileno (stream), TCSAFLUSH, &new) != 0)
    return -1;

  /* Parolayı oku */
  nread = getline (lineptr, n, stream);

  /* Uçbirimi eski durumuna getir. */
  (void) tcsetattr (fileno (stream), TCSAFLUSH, &old);

  return nread;
}

Örnekte getline aynı bağımsız değişkenleri almaktadır (bkz, Satır Yönlenimli Girdi); yazılımcı uygun istemi göstermelidir.

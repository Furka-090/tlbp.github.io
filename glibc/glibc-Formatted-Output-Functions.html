12.12.7. Biçimli Çıktı İşlevleri

Bu bölümde printf ve ilgili işlevlerin nasıl çağrıldığı anlatılmıştır. Bu işlevler stdio.h başlık dosyasında bildirilmiştir. Bu işlevlerin değişken sayıda bağımsız değişken almalarından dolayı, kullanılmadan önce ilk örnekleri ile bildirilmelidir. Şüphesiz bunu yapmanın en kolay yolu bu işlevlerin bildirildiği stdio.h başlık dosyasının kaynak dosyasında içerilmesidir.

      işlev
#include <stdio.h>
int printf(const char *template, …)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe corrupt | AC-Unsafe mem lock corrupt

printf işlevi template biçim dizgesinin denetimi altında isteğe bağlı bağımsız değişkenleri standart çıktıya basar ve basılan karakterlerin sayısı ile döner; bir çıktılama hatası oluşursa negatif değerle döner.

      işlev
#include <wchar.h>
int wprintf(const wchar_t *template, …)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe corrupt heap | AC-Unsafe mem lock corrupt

wprintf işlevi template geniş biçim dizgesinin denetimi altında isteğe bağlı bağımsız değişkenleri standart çıktıya basar ve basılan geniş karakterlerin sayısı ile döner; bir çıktılama hatası oluşursa negatif değerle döner.

      işlev
#include <stdio.h>
int fprintf(FILE *stream, const char *template, …)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe corrupt heap | AC-Unsafe mem lock corrupt

Bu işlev printf gibidir, tek farkla, çıktıyı standart çıktıya değil stream akımına yazar.

      işlev
#include <stdio.h>
#include <wchar.h>
int fwprintf(FILE *stream, const wchar_t *template, …)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe corrupt heap | AC-Unsafe mem lock corrupt

Bu işlev wprintf gibidir, tek farkla, çıktıyı standart çıktıya değil stream akımına yazar.

      işlev
#include <stdio.h>
int sprintf(char *s, const char *template, …)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe heap | AC-Unsafe mem

Bu işlev printf gibidir, tek farkla, çıktıyı standart çıktıya değil s dizisine yazar. Dizge boş karakterle sonlandırılır.

sprintf işlevi sonlandırıcı boş karakter hariç s dizisine yazılan karakterlerin sayısı ile döner.

Bu işlevin davranışı, birbirini kapsayan nesneler arasında kopyalama yapılırsa, örneğin, %s belirtecinin bağımsız değişkeni olarak s dizgesi verilmişse, tanımsızdır. Bkz. Dizilerin ve Dizgelerin Kopyalanması.

Uyarı

sprintf işlevi tehlikeli olabilir. s dizgesi için tahsis edilen yerin biçim dizgesinin ürettiği dizge için yetersiz olabilme olasılığı vardır. Alan genişliği bağımsız değişkeninin en küçük alan genişliğini belirttiği ve kimi zaman bu genişliğin aşılabildiği unutulmamalıdır.

Bu sorundan kaçımak için aşağıda açıklanacak olan snprintf veya asprintf işlevleri kullanılabilir.

      işlev
#include <wchar.h>
int swprintf(wchar_t *ws, size_t size, const wchar_t *template, …)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe heap | AC-Unsafe mem

Bu işlev wprintf gibidir, tek farkla, çıktıyı standart çıktıya değil ws geniş karakter dizisine yazar. Dizge boş geniş karakterle sonlandırılır. size bağımsız değişkeni ile üretilecek en büyük karakter sayısı belirtilir. Sonlandırıcı boş karakter de bu sınırın içinde olduğundan ws dizgesi için en az size geniş karakterlik yer ayrılmalıdır.

swprintf işlevi sonlandırıcı boş geniş karakter hariç ws geniş karakter dizisine yazılan karakterlerin sayısı ile döner. Çıktı belirtilen tampona sığmazsa negatif değer ile döner. Bu davranışı ile işlev snprintf işlevinden farklıdır.

Bilginize

Benzeri olan dar yönlenimli akım işlevleri daha az bağımsız değişken alır. swprintf işlevi aslında snprintf işlevinin karşılığıdır. sprintf işlevi tehlikeli olabildiğinden, ISO C komitesi aynı yanlışı tekrar yapmaktan kaçınmalı ve sprintf işlevinin tam karşılığı olan bir işlevi tanımlamama kararı almalıdır.

      işlev
#include <stdio.h>
int snprintf(char *s, size_t size, const char *template, …)
Evresel Güvenilirlik: | MT-Safe locale | AS-Unsafe heap | AC-Unsafe mem

snprintf işlevi, sprintf gibidir, tek farkla, size bağımsız değişkeni ile üretilecek en büyük karakter sayısı belirtilir. Sonlandırıcı boş karakter de bu sınırın içinde olduğundan sdizgesi için en az size karakterlik yer tahsis edilmelidir.

snprintf işlevi sonlandırıcı boş geniş karakter hariç s karakter dizisine yazılan karakterlerin sayısı ile döner. Bu değer size bayta eşit ya da ondan büyükse sonucun tüm karakterleri s dizisine yazılamamış demektir. Bu durumda daha büyük çıktı dizisi ile çağrı yinelenmelidir. Aşağıda bunun yapıldığı bir örnek vardır:

Örnek: 
/* 'name' isminde ve 'value' değerinde
    bir değişkenin değerini açıklayan bir ileti oluştur. */
char *
make_message (char *name, char *value)
{
  /* Nasılsa 100 karakterden daha fazla yere ihtiyaç olmaz. */
  size_t size = 100;
  char *buffer = xmalloc (size);
  /* Tahsis edilen alana iletiyi yazmaya çalış. */
  int buflen = snprintf (buffer, size, "value of %s is %s",
		         name, value);
  if (! (0 <= buflen && buflen < SIZE_MAX))
    fatal ("integer overflow");
  if (buflen >= size)
    {
      /* Artık ne kadar yere ihtiyaç olduğu biliniyor.
         O halde tamponu yeniden ayır. */
      size = buflen;
      size++;
      buffer = xrealloc (buffer, size);

      /* Yeniden dene. */
      snprintf (buffer, size, "value of %s is %s",
		name, value);
    }
  /* Son çağrı çalıştığından dizge ile dön. */
  return buffer;
}

Aslında, asprintf işlevini kullanmak çok daha kolaydır.

Dikkat

GNU C Kütüphanesinin 2.1 öncesi sürümlerinde dönüş değeri, sonlandırıcı boş karakter hariç saklanan karakterlerin sayısıydı. s dizisinde yeterli yer yoksa -1 dönüyordu. Bu, ISO C99 standardına uyum sırasında değiştirildi.
